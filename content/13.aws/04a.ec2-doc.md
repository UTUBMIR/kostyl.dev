# AWS EC2 CLI — Довідник команд

Цей довідник охоплює всі важливі `aws ec2` команди, згруповані за категоріями. Для кожної команди наведено опис ключових параметрів та очікуваний вивід терміналу.

::note
Перед використанням переконайтеся, що AWS CLI налаштований: `aws configure` або змінні середовища `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_DEFAULT_REGION`. Більшість команд потребують `--region` — зручно встановити через `export AWS_DEFAULT_REGION=eu-central-1`.
::

## EC2 Instances — управління instances

### run-instances

Запускає один або кілька EC2 instances із вказаним AMI. Це основна команда для створення нових серверів.

::field-group

::field{name="--image-id" type="string" required="true"}
AMI ID для запуску instance. Формат `ami-xxxxxxxxxxxxxxxxx`. Прив'язаний до регіону — один і той самий AMI у різних регіонах має різний ID.
::

::field{name="--instance-type" type="string" required="true"}
Тип instance: `t3.micro`, `t3.medium`, `m6i.large` тощо. Визначає кількість vCPU, RAM та мережеву продуктивність.
::

::field{name="--key-name" type="string"}
Ім'я Key Pair для SSH-підключення. Без нього не можна підключитись через SSH (якщо не використовується Instance Connect або SSM).
::

::field{name="--security-group-ids" type="list"}
Список ID Security Groups через пробіл: `sg-111 sg-222`. Визначає правила мережевого доступу.
::

::field{name="--subnet-id" type="string"}
ID підмережі VPC для запуску instance. Визначає зону доступності (AZ). Якщо не вказано — AWS вибирає підмережу за замовчуванням.
::

::field{name="--count" type="string"}
Кількість instances для запуску у форматі `мін:макс`, наприклад `1:3`. Або просто число для точної кількості.
::

::field{name="--user-data" type="string"}
Bootstrap-скрипт, що виконується при першому запуску. Передається як `file://path/to/script.sh` або як inline base64-рядок.
::

::field{name="--iam-instance-profile" type="string"}
IAM Instance Profile для прикріплення ролі: `Name=my-profile` або `Arn=arn:aws:iam::...`. Надає instance доступ до AWS-сервісів без hardcode ключів.
::

::field{name="--tag-specifications" type="string"}
Теги для instance та EBS томів прямо при запуску: `'ResourceType=instance,Tags=[{Key=Name,Value=my-server}]'`.
::

::field{name="--block-device-mappings" type="json"}
Конфігурація EBS томів: розмір, тип, шифрування. Наприклад, збільшити root том до 30 GB.
::

::

::terminal-preview{title="aws ec2 run-instances — запуск Ubuntu instance"}

<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 run-instances \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--image-id ami-0a1b2c3d4e5f67890 \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--instance-type t3.micro \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--key-name ec2-lab-key \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--security-group-ids sg-0a1b2c3d4e5f67890 \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=dotnet-api-server}]' \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--query "Instances[0].InstanceId" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--output text --region eu-central-1</strong></div>
<div class="line"><span class="text-green-400">i-0a1b2c3d4e5f67890</span></div>

::

::tip
`--query "Instances[0].InstanceId" --output text` одразу повертає чистий ID без JSON — зручно для збереження в змінну: `INSTANCE_ID=$(aws ec2 run-instances ... --output text)`.
::

### describe-instances

Повертає інформацію про один або кілька instances: стан, тип, IP-адреси, Security Groups, теги тощо.

::field-group

::field{name="--instance-ids" type="list"}
Список Instance ID для фільтрації: `i-111 i-222`. Без цього параметра повертає всі instances у регіоні.
::

::field{name="--filters" type="list"}
Фільтри у форматі `Name=key,Values=val1,val2`. Популярні фільтри: `instance-state-name` (running/stopped), `tag:Name`, `instance-type`, `image-id`.
::

::field{name="--query" type="string"}
JMESPath вираз для форматування виводу. Часто використовується для вилучення конкретних полів.
::

### start-instances / stop-instances / reboot-instances

Змінюють стан одного або кількох instances. `stop` зупиняє instance зі збереженням EBS (обчислення не тарифікуються). `reboot` перезавантажує без зміни IP та фізичного хосту.

::field-group

::field{name="--instance-ids" type="list" required="true"}
Один або кілька Instance ID: `i-111 i-222`. Усі вказані instances отримують нову команду одночасно.
::

::field{name="--hibernate" type="boolean"}
Лише для `stop-instances`. Зберігає вміст RAM на EBS-диск. Instance відновиться з того самого стану (вимагає попереднього налаштування Hibernate).
::

::

### terminate-instances

**Незворотньо** видаляє instance. За замовчуванням root EBS том також видаляється. Перед виконанням переконайтеся, що вибрали правильний Instance ID.

::field-group

::field{name="--instance-ids" type="list" required="true"}
Instance ID для видалення. Можна передати кілька: `i-111 i-222 i-333`.
::

::

::terminal-preview{title="aws ec2 terminate-instances"}

<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 terminate-instances --instance-ids i-0a1b2c3d4e5f67890 --region eu-central-1</strong></div>
<div class="line">{</div>
<div class="line">    "TerminatingInstances": [{</div>
<div class="line">        "InstanceId": "i-0a1b2c3d4e5f67890",</div>
<div class="line">        "CurrentState": { "Name": <span class="text-yellow-400">"shutting-down"</span> },</div>
<div class="line">        "PreviousState": { "Name": <span class="text-green-400">"running"</span> }</div>
<div class="line">    }]</div>
<div class="line">}</div>

::

::caution
`terminate-instances` — незворотна операція. Після переходу в `terminated` instance та root EBS том не можна відновити. Завжди перевіряйте `--instance-ids` перед виконанням.
::

### wait

Блокує виконання скрипту поки instance не досягне цільового стану. Незамінний в автоматизації — дозволяє уникнути polling вручну та `sleep`.

::field-group

::field{name="instance-running" type="subcommand"}
Чекати поки instance перейде у стан `running`. Виконується після `run-instances` або `start-instances`.
::

::field{name="instance-stopped" type="subcommand"}
Чекати поки instance перейде у стан `stopped`. Корисно перед зміною типу instance або створенням AMI.
::

::field{name="instance-status-ok" type="subcommand"}
Чекати не лише `running`, але й успішне проходження обох status checks (system check + instance check). Більш надійний варіант перед SSH-підключенням.
::

::field{name="instance-terminated" type="subcommand"}
Чекати повного видалення instance. Потрібно перед звільненням Elastic IP або Security Group.
::

::field{name="--instance-ids" type="list" required="true"}
Instance ID, для якого очікуємо стан.
::

::

::terminal-preview{title="aws ec2 wait instance-running — блокуючий старт у скрипті"}

<div class="line"><span class="opacity-40">$</span> <strong>INSTANCE_ID=$(aws ec2 run-instances \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--image-id ami-0a1b2c3d4e5f67890 --instance-type t3.micro \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--query "Instances[0].InstanceId" --output text --region eu-central-1)</strong></div>
<div class="line"></div>
<div class="line"><span class="opacity-40">$</span> <strong>echo "Waiting for instance $INSTANCE_ID to start..."</strong></div>
<div class="line">Waiting for instance i-0a1b2c3d4e5f67890 to start...</div>
<div class="line"></div>
<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 wait instance-status-ok \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--instance-ids $INSTANCE_ID --region eu-central-1</strong></div>
<div class="line"><span class="opacity-40"># ~90 секунд очікування...</span></div>
<div class="line"></div>
<div class="line"><span class="opacity-40">$</span> <strong>echo "Instance is ready!"</strong></div>
<div class="line"><span class="text-green-400">Instance is ready!</span></div>

::

## AMI — Amazon Machine Images

### describe-images

Шукає AMI у поточному регіоні за фільтрами. Основне використання — знайти актуальний AMI ID для автоматизованих скриптів.

::field-group

::field{name="--owners" type="list" required="true"}
Власник AMI. `099720109477` — Canonical (Ubuntu), `amazon` — офіційні Amazon AMI, `self` — ваші Custom AMI, `aws-marketplace` — Marketplace.
::

::field{name="--filters" type="list"}
Фільтри для пошуку: `Name=name,Values=*pattern*`, `Name=state,Values=available`, `Name=architecture,Values=x86_64`.
::

::field{name="--image-ids" type="list"}
Знайти конкретні AMI за ID. Корисно для перевірки існування або отримання деталей.
::

::field{name="--query" type="string"}
Зазвичай використовується `sort_by(Images, &CreationDate)[-1].ImageId` — повертає ID найновішого AMI з результатів фільтрації.
::

::

::terminal-preview{title="aws ec2 describe-images — актуальний Ubuntu 24.04 AMI"}

<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 describe-images \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--owners 099720109477 \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--filters "Name=name,Values=ubuntu/images/hvm-ssd-gp3/ubuntu-noble-24.04-amd64-server-*" \</strong></div>
<div class="line"><span class="opacity-40">></span>             <strong>"Name=state,Values=available" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--query "sort_by(Images, &amp;CreationDate)[-1].ImageId" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--output text --region eu-central-1</strong></div>
<div class="line"><span class="text-green-400">ami-0a1b2c3d4e5f67890</span></div>

::

<div class="line">-----------------------------------------------------------------</div>

::

::tip
Ніколи не хардкодьте AMI ID у скриптах автоматизації — вони змінюються при виході нових версій. Завжди запитуйте актуальний ID через `describe-images` з відповідними фільтрами.

### create-image

Створює Custom AMI зі знімку запущеного або зупиненого instance. AMI зберігає повний стан EBS-дисків.

::field-group

::field{name="--instance-id" type="string" required="true"}
Instance ID, з якого створюється AMI.
::

::field{name="--name" type="string" required="true"}
Ім'я нового AMI. Унікальне в межах акаунту та регіону.
::

::field{name="--description" type="string"}
Опис AMI для документування вмісту (версія .NET, встановлені пакети тощо).
::

::field{name="--no-reboot" type="boolean"}
Не перезавантажувати instance під час створення AMI. Швидше, але може призвести до неконсистентного стану файлової системи. Для продакшн — краще без `--no-reboot`.
::

::

::terminal-preview{title="aws ec2 create-image — Custom AMI з запущеного instance"}

<div class="line"><span class="opacity-40">$</span> <strong>AMI_ID=$(aws ec2 create-image \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--instance-id i-0a1b2c3d4e5f67890 \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--name "dotnet-api-ready-v1.2.0" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--description "Ubuntu 24.04 + .NET 8 SDK + nginx 1.24" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--query ImageId --output text --region eu-central-1)</strong></div>
<div class="line"></div>
<div class="line"><span class="opacity-40">$</span> <strong>echo "AMI: $AMI_ID"</strong></div>
<div class="line">AMI: <span class="text-green-400">ami-0c2d3e4f5a678901b</span></div>
<div class="line"></div>
<div class="line"><span class="opacity-40"># Очікуємо поки AMI стане доступним</span></div>
<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 wait image-available --image-ids $AMI_ID --region eu-central-1</strong></div>
<div class="line"><span class="text-green-400"># 3-15 хвилин... AMI готовий</span></div>

### deregister-image

Видаляє реєстрацію AMI — після цього неможливо запустити нові instances з цього AMI. Вже запущені instances продовжують працювати. EBS snapshots, пов'язані з AMI, **не видаляються** автоматично — потрібно видалити вручну.

::field-group

::field{name="--image-id" type="string" required="true"}
ID AMI для видалення.
::

::

::terminal-preview{title="aws ec2 deregister-image — видалення старого AMI та його snapshot"}

<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 deregister-image --image-id ami-0c2d3e4f5a678901b --region eu-central-1</strong></div>
<div class="line"><span class="text-green-400">(no output — success)</span></div>
<div class="line"></div>
<div class="line"><span class="opacity-40"># Знайти та видалити пов'язані snapshots</span></div>
<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 describe-snapshots \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--owner-ids self \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--filters "Name=description,Values=*ami-0c2d3e4f5a678901b*" \</strong></div>
<div class="line"><span class="opacity-40">></span>     <strong>--query "Snapshots[*].SnapshotId" --output text --region eu-central-1</strong></div>
<div class="line">snap-0d3e4f5a678901bc2</div>
<div class="line"></div>
<div class="line"><span class="opacity-40">$</span> <strong>aws ec2 delete-snapshot --snapshot-id snap-0d3e4f5a678901bc2 --region eu-central-1</strong></div>
<div class="line"><span class="text-green-400">(no output — success)</span></div>

::

### copy-image

Копіює AMI між регіонами або в межах того самого регіону. Необхідно для мультирегіональних Auto Scaling груп або аварійного відновлення.

::field-group

::field{name="--source-image-id" type="string" required="true"}
ID AMI у вихідному регіоні.
::

::field{name="--source-region" type="string" required="true"}
Регіон, де знаходиться AMI-джерело.
::

::field{name="--name" type="string" required="true"}
Ім'я копії у цільовому регіоні.
::

::field{name="--region" type="string" required="true"}
Цільовий регіон для копії.
::

::

::terminal-preview{title="aws ec2 copy-image — копіювання AMI з eu-central-1 до us-east-1"}

<div class="line"><span class="text-green-400">ami-0e4f5a678901bc2d3</span></div>

::

## Key Pairs — SSH ключі

### create-key-pair

Створює нову Key Pair та повертає **приватний ключ** у JSON-відповіді. Це єдиний момент, коли AWS показує приватний ключ — одразу збережіть його у файл.

::field-group

::field{name="--key-name" type="string" required="true"}

### Швидка автоматизація — змінні середовища для роботи

```bash
# Базові змінні для роботи в скриптах

export ALLOC_ID="eipalloc-0a1b2c3d4e5f67890"
```

### Отримати публічний IP instance

```bash
aws ec2 describe-instances \
    --instance-ids $INSTANCE_ID \
    --query "Reservations[0].Instances[0].PublicIpAddress" \
    --output text --region $REGION
```

### Знайти default VPC і підмережу

```bash
VPC_ID=$(aws ec2 describe-vpcs \
    --filters "Name=isDefault,Values=true" \
    --query "Vpcs[0].VpcId" --output text --region $REGION)

SUBNET_ID=$(aws ec2 describe-subnets \
    --filters "Name=vpc-id,Values=$VPC_ID" \
              "Name=defaultForAz,Values=true" \
    --query "Subnets[0].SubnetId" --output text --region $REGION)
```

### Зупинити і видалити instance + EIP + Security Group

```bash
# Повне очищення ресурсів після lab-роботи
REGION="eu-central-1"

# 1. Від'єднати та звільнити EIP
ASSOC_ID=$(aws ec2 describe-addresses \
    --allocation-ids $ALLOC_ID \
    --query "Addresses[0].AssociationId" --output text --region $REGION)
aws ec2 disassociate-address --association-id $ASSOC_ID --region $REGION
aws ec2 release-address --allocation-id $ALLOC_ID --region $REGION

# 2. Видалити instance
aws ec2 terminate-instances --instance-ids $INSTANCE_ID --region $REGION
aws ec2 wait instance-terminated --instance-ids $INSTANCE_ID --region $REGION

# 3. Видалити Security Group (лише після того, як instance terminated)
aws ec2 delete-security-group --group-id $SG_ID --region $REGION

echo "Cleanup complete!"
```

### Список усіх instances у регіоні з тегами Name та станом

```bash
aws ec2 describe-instances \
    --query "Reservations[*].Instances[*].{ID:InstanceId,Name:Tags[?Key=='Name'].Value|[0],State:State.Name,Type:InstanceType,IP:PublicIpAddress}" \
    --output table --region $REGION
```
