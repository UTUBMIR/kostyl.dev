# План вивчення структур у C++ — серія статей

## Огляд серії

Структури — тема, яка вимагає повного, поступового розкриття.
Серія складається з **чотирьох статей** (~1000 рядків кожна).

## Стаття 33 — `struct`: агрегування даних

**Файл:** `33.struct.md`
**Орієнтовний обсяг:** ~1000 рядків
**Залежності:** `05.data-types-variables.md`

### Навчальні цілі
- Пояснити проблему розрізнених даних (паралельні масиви, три параметри замість одного)
- Ввести поняття «сутність» і «атрибут» без згадки ООП
- Навчити синтаксису `struct`, полям, `;` після `}`
- Розкрити агрегатну ініціалізацію, designated initializers (C++20), default member init
- Показати оператор `.` та `->`, `::debugger-view`
- Пояснити `sizeof` структури, вирівнювання (padding), `::memory-view`

### Структура статті
1. Hook: три паралельних масиви — синхронізація руками
2. «Програмування як моделювання» — сутності, атрибути (без ООП)
3. Визначення агрегатного типу; «оголошення типу» vs «визначення змінної»
4. Синтаксис: `struct`, поля, `;`, PascalCase/camelCase
5. Ініціалізація (три підходи: присвоєння, агрегатна, designated C++20)
6. Оператор `.` і `->`, `::debugger-view`
7. `sizeof` і вирівнювання: чому `struct { char c; int n; }` — 8 байт, не 5
8. `::memory-view` для візуалізації padding
9. Практика (3 рівні, accordion)
10. Резюме

## Стаття 34 — Структури у функціях і повернення значень

**Файл:** `34.struct-functions.md`
**Орієнтовний обсяг:** ~1000 рядків
**Залежності:** `33.struct.md`, `12.functions-basics.md`, `15.pointers-basics.md`

### Навчальні цілі
- Детально порівняти три способи передачі (за значенням, `const&`, `*`)
- Пояснити вартість копіювання великих структур
- Розкрити NRVO (Named Return Value Optimization)
- Показати фабричні функції як патерн (замість конструктора)
- Функції-запити (`const&`) і функції-мутатори (`&` або `*`)

### Структура статті
1. Hook: функція з шістьма параметрами — «а якщо додати поле?»
2. Передача за значенням: стек, копія, вартість для `std::string`-полів
3. Передача `const T&`: семантика «лише читати»
4. Передача `T&`: мутатор, «змінити оригінал»
5. Передача `T*`: відмінності від посилання, nullptr-захист
6. Повернення структури: NRVO
7. Патерн «фабрична функція»: `createStudent(...)` замість конструктора
8. Патерн «трансформація»: `translate(const Point3D&, dx, dy, dz)`
9. Патерн «агрегування з перевіркою»: валідація перед створенням
10. Практика та резюме

## Стаття 36 — Патерни використання struct і межі застосування

**Файл:** `36.struct-patterns.md`
**Орієнтовний обсяг:** ~1000 рядків
**Залежності:** `35.struct-arrays.md`

### Навчальні цілі
- Практичні патерни: Record, Value Object, Result, Named Parameters
- Семантика значення vs семантика посилання
- `struct` vs `class` — технічна різниця
- Наприкінці: методи в `struct` — можливо, але чому це семантична помилка

### Структура статті
1. Патерн «Record» — struct як чистий контейнер даних (POD)
2. Патерн «Value Object» — `Point`, `Color`, `Money`
3. Патерн «Result» — `struct ParseResult { bool ok; int value; std::string error; }`
4. Патерн «Named Parameters» — `struct SortOptions { bool ascending; bool stable; }`
5. Семантика значення: копіювання при присвоєнні
6. `struct` vs `class`: єдина технічна різниця — доступ за замовчуванням
7. **Структури з методами** — демонстрація, що це технічно можливо
8. **Позиція Страуструпа:**
   - `struct` = plain data, aggregate, без інваріантів
   - Як тільки з'являється поведінка — з'являються інваріанти
   - `class` захищає їх через `private`; `struct` не може цього гарантувати
   - Цитата з «The C++ Programming Language»
   - Код із `struct`+методи вводить читача в оману щодо намірів автора
9. Підсумок серії та анонс `class`
