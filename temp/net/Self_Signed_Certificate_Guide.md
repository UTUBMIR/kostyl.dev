## Що таке самопідписаний сертифікат і навіщо він потрібен?

**Самопідписаний сертифікат (Self-Signed Certificate)** — це SSL/TLS-сертифікат, який підписаний своїм власним приватним ключем, а не ключем довіреного Центру сертифікації (CA).

-   **Звичайний сертифікат:** Довіра ґрунтується на тому, що його підписав авторитетний CA (напр., Let's Encrypt, DigiCert), який перевірив власника домену. Ваш браузер довіряє цим CA за замовчуванням.
-   **Самопідписаний сертифікат:** Браузер не знає, хто створив цей сертифікат, і не може перевірити його автентичність. Тому за замовчуванням він **не довіряє** таким сертифікатам і показує попередження про небезпеку.

**Навіщо він потрібен?**

-   **Локальна розробка:** Дозволяє запускати ваш веб-додаток по HTTPS на `localhost` або інших локальних доменах (`mysite.local`). Це важливо для тестування функціоналу, який працює тільки в безпечному контексті (напр., Service Workers, геолокація, Web Crypto API).
-   **Внутрішні (інтранет) сервіси:** Для внутрішніх серверів компанії, де публічна довіра не потрібна, а співробітники можуть вручну додати сертифікат до довірених.
-   **Тестування:** Швидкий спосіб налаштувати шифрування без необхідності купувати сертифікат або налаштовувати Let's Encrypt.

**Головне правило:** Ніколи не використовуйте самопідписані сертифікати для публічних, продакшн-сайтів!

## Інструменти: OpenSSL

**OpenSSL** — це потужна криптографічна бібліотека з відкритим кодом, яка є стандартом для роботи з сертифікатами та ключами. Вона доступна на більшості операційних систем:

-   **macOS та Linux:** Зазвичай встановлена за замовчуванням.
-   **Windows:** Можна встановити через [Chocolatey](https://chocolatey.org/packages/openssl), [WSL](https://learn.microsoft.com/en-us/windows/wsl/install) або завантаживши бінарні файли з [офіційного сайту](https://www.openssl.org/source/).

Перевірити, чи встановлено OpenSSL, можна командою в терміналі:

```bash
openssl version
```

## Крок 1: Створення приватного ключа та сертифіката

Ми можемо створити приватний ключ та самопідписаний сертифікат однією командою.

Відкрийте термінал і виконайте:

```bash
openssl req -x509 -newkey rsa:2048 -nodes -keyout localhost.key -out localhost.crt -days 365 -subj "/C=UA/ST=Kyiv/L=Kyiv/O=LocalDev/OU=DevTeam/CN=localhost" -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
```

Давайте розберемо цю команду:

-   `openssl req`: Команда для роботи із запитами на підпис сертифіката (CSR).
-   `-x509`: Вказує, що ми хочемо створити самопідписаний сертифікат, а не запит (CSR).
-   `-newkey rsa:2048`: Генерує новий приватний ключ RSA довжиною 2048 біт.
-   `-nodes`: (No DES) Не шифрувати приватний ключ паролем. Це зручно для локальної розробки, щоб сервер міг читати ключ без пароля. **Для продакшн-ключів пароль є обов'язковим!**
-   `-keyout localhost.key`: Зберегти приватний ключ у файл `localhost.key`.
-   `-out localhost.crt`: Зберегти сертифікат у файл `localhost.crt`.
-   `-days 365`: Встановити термін дії сертифіката на 365 днів.
-   `-subj "/C=UA/ST=Kyiv/..."`: Заповнює інформацію про власника сертифіката в неінтерактивному режимі.
    -   `CN=localhost`: **Common Name**. Найважливіше поле, має збігатися з доменом вашого сайту.
-   `-addext "subjectAltName=DNS:localhost,IP:127.0.0.1"`: **Subject Alternative Name (SAN)**. Це критично важливе розширення. Сучасні браузери вимагають, щоб домен був вказаний не тільки в `CN`, але і в `SAN`. Ми додаємо `localhost` як DNS-ім'я та `127.0.0.1` як IP-адресу.

Після виконання цієї команди у вас з'являться два файли:

-   `localhost.key`: Ваш приватний ключ. **Тримайте його в секреті!**
-   `localhost.crt`: Ваш самопідписаний сертифікат.

## Крок 2: Налаштування веб-сервера

Тепер, коли у вас є ключ і сертифікат, потрібно налаштувати ваш локальний веб-сервер, щоб він їх використовував.

### Приклад для .NET (Kestrel)

У проектах ASP.NET Core налаштувати HTTPS для локального сервера Kestrel можна через файл `appsettings.Development.json`.

Спочатку переконайтеся, що ви створили `.pfx` файл, як описано вище.

```json
{
    "Kestrel": {
        "Endpoints": {
            "Http": {
                "Url": "http://localhost:5000"
            },
            "Https": {
                "Url": "https://localhost:5001",
                "Certificate": {
                    "Path": "path/to/your/localhost.pfx",
                    "Password": "your_pfx_password"
                }
            }
        }
    }
}
```

**Пояснення:**

-   Ми визначаємо дві кінцеві точки (endpoints): одну для HTTP, іншу для HTTPS.
-   В секції `Https` ми вказуємо шлях до нашого `.pfx` файлу та пароль, який ви встановили при його створенні.

Тепер при запуску вашого .NET додатку Kestrel буде слухати порт 5001 і використовувати ваш самопідписаний сертифікат для HTTPS-з'єднань.

### Приклад для Nginx

Якщо ви використовуєте Nginx як реверс-проксі для вашого локального додатку, ви можете налаштувати його для обробки HTTPS.

Вам знадобляться файли `localhost.key` та `localhost.crt`.

Приклад конфігурації Nginx (`nginx.conf` або в `sites-available`):

```nginx
server {
    listen 80;
    listen 443 ssl;
    server_name localhost;

    # Шляхи до сертифіката та ключа
    ssl_certificate /path/to/your/localhost.crt;
    ssl_certificate_key /path/to/your/localhost.key;

    # Рекомендовані налаштування SSL
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers on;

    # Перенаправлення HTTP на HTTPS
    if ($scheme != "https") {
        return 301 https://$host$request_uri;
    }

    location / {
        # Проксування на ваш бекенд-додаток
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Крок 3: Додавання сертифіката до довірених (щоб уникнути помилок)

Щоб браузер не показував попередження "Your connection is not private", потрібно додати ваш самопідписаний сертифікат (`localhost.crt`) до списку довірених кореневих сертифікатів у вашій операційній системі.

### Інструкції для Windows

1.  **Відкрийте консоль керування сертифікатами:**

    -   Натисніть `Win + R`, введіть `mmc` і натисніть Enter.
    -   У вікні, що відкрилося, виберіть `File -> Add/Remove Snap-in...`.
    -   Зі списку доступних оснасток виберіть `Certificates` і натисніть `Add >`.
    -   У новому вікні виберіть `Computer account` і натисніть `Next`, потім `Finish` і `OK`.

2.  **Імпортуйте сертифікат:**

    -   У лівій панелі розкрийте `Certificates (Local Computer) -> Trusted Root Certification Authorities`.
    -   Клацніть правою кнопкою миші на папці `Certificates`, виберіть `All Tasks -> Import...`.
    -   Натисніть `Next`, потім `Browse...` і виберіть ваш файл `localhost.crt`.
    -   Переконайтеся, що сертифікат розміщується у сховищі `Trusted Root Certification Authorities`.
    -   Натисніть `Next` і `Finish`.

3.  **Перезапустіть браузер:** Повністю закрийте і знову відкрийте ваш браузер. Попередження має зникнути.

### Інструкції для macOS

1.  **Відкрийте Keychain Access:**

    -   Знайдіть `Keychain Access` через Spotlight (`Cmd + Space`).

2.  **Імпортуйте сертифікат:**

    -   У лівій панелі виберіть `System` в розділі `Keychains`.
    -   Перетягніть ваш файл `localhost.crt` у вікно Keychain Access.
    -   Введіть пароль адміністратора.

3.  **Зробіть сертифікат довіреним:**

    -   Знайдіть ваш сертифікат у списку (зазвичай за `CN`, тобто `localhost`).
    -   Двічі клацніть на ньому, щоб відкрити деталі.
    -   Розкрийте секцію `Trust`.
    -   У полі `When using this certificate:` виберіть `Always Trust`.
    -   Закрийте вікно. Вам може знадобитися знову ввести пароль.

4.  **Перезапустіть браузер.**

### Інструкції для Linux (Ubuntu/Debian)

Система довіри в Linux децентралізована. Firefox має власне сховище сертифікатів, тоді як Chrome та інші додатки використовують системне.

#### Для Chrome та інших системних додатків:

1.  **Скопіюйте сертифікат у системну папку:**

    ```bash
    sudo cp localhost.crt /usr/local/share/ca-certificates/localhost.crt
    ```

2.  **Оновіть сховище сертифікатів:**

    ```bash
    sudo update-ca-certificates
    ```

    Ви повинні побачити повідомлення, що один сертифікат було додано.

3.  **Перезапустіть браузер.**

#### Для Firefox:

1.  **Відкрийте налаштування:**

    -   Перейдіть у `Settings -> Privacy & Security`.
    -   Прокрутіть до секції `Certificates` і натисніть `View Certificates...`.

2.  **Імпортуйте сертифікат:**

    -   У вкладці `Authorities` натисніть `Import...`.
    -   Виберіть ваш файл `localhost.crt`.

## Обмеження та найкращі практики

-   **Тільки для розробки:** Ще раз наголошуємо: ніколи не використовуйте самопідписані сертифікати на публічних сайтах. Користувачі побачать страшне попередження про небезпеку і не зможуть довіряти вашому сайту.
-   **Не діліться приватним ключем:** Файл `localhost.key` має залишатися тільки на вашому комп'ютері.
-   **Командна робота:** Якщо у вашій команді кільком розробникам потрібен доступ до локального HTTPS, краще створити свій власний "локальний" CA. Ви можете створити один кореневий сертифікат CA, додати його до довірених на всіх комп'ютерах команди, а потім використовувати його для підпису сертифікатів для кожного розробника. Інструменти, як-от `mkcert`, автоматизують цей процес.
-   **Термін дії:** Встановлюйте адекватний термін дії (`-days`). 365 днів є розумним вибором для локального сертифіката.

## Висновок

Створення самопідписаного сертифіката — це простий та ефективний спосіб налаштувати HTTPS для локальної розробки. Це дозволяє вам працювати в середовищі, максимально наближеному до продакшну, і тестувати функціонал, що вимагає безпечного з'єднання.

Головне — пам'ятати про сферу їх застосування і ніколи не використовувати їх для реальних, публічних веб-сайтів, де довіра користувачів та верифікація через довірений CA є абсолютно необхідними. - У вікні, що з'явиться, поставте галочку `Trust this CA to identify websites`. - Натисніть `OK`.

Для деяких систем (наприклад, .NET) може знадобитися сертифікат у форматі `.pfx` (або `.p12`), який об'єднує ключ і сертифікат в один файл. Створити його можна командою:

```bash
openssl pkcs12 -export -out localhost.pfx -inkey localhost.key -in localhost.crt
```

Вас попросять ввести пароль для захисту `.pfx` файлу.
