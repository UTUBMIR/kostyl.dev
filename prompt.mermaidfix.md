# Документація: Валідація та виправлення Mermaid-діаграм

Цей документ описує процес автоматизованої перевірки синтаксису та виправлення помилок у Mermaid-діаграмах всередині Markdown-файлів за допомогою спеціальних скриптів.

---

## Огляд системи

Система складається з двох основних інструментів:

1. **`validate_mermaid.py`** — скрипт для виявлення помилок у синтаксисі діаграм.
2. **`fix_mermaid.py`** — скрипт для автоматичного виправлення типових помилок (лапки, несумісні стилі тощо).

### Ключові можливості

- ✅ Автоматичне вилучення всіх Mermaid-блоків з Markdown-файлу.
- ✅ Перевірка кожного блоку через офіційний `mermaid-cli`.
- ✅ Детальний звіт про помилки із зазначенням конкретних рядків.
- ✅ Автоматизоване виправлення синтаксису для Sequence-діаграм.
- ✅ Підтримка кирилиці та спеціальних символів у діаграмах.

---

## Скрипт 1: `validate_mermaid.py`

### Призначення

Аналізує Markdown-файл, знаходить усі блоки ` ```mermaid ` і перевіряє їх на валідність.

### Використання

```bash
# Перевірити конкретний файл
python scripts/validate_mermaid.py content/07.tools/02.kubernetes/02.kubernetes-architecture.md
```

### Приклад звіту

```text
Validating 9 blocks in content/02.kubernetes-architecture.md...

--- MERMAID VALIDATION REPORT ---
Block 1 (Lines 131-151): ❌ INVALID
Error Details:
Error: Parse error on line 18: ... style Auth fill:#f59e0b ... got 'TXT'

Block 2 (Lines 187-207): ✅ VALID
...
Found errors in some Mermaid diagrams.
```

---

## Скрипт 2: `fix_mermaid.py`

### Призначення

Автоматично виправляє поширені помилки, які часто роблять ШІ або розробники при написанні Mermaid-коду.

### Що він виправляє

1. **Видалення `style` із Sequence Diagrams**: Mermaid не підтримує flowchart-стилізацію (`style ID fill:#...`) у діаграмах послідовності.
2. **Автоматичне цитування**: Додає подвійні лапки до тексту вузлів та переходів, якщо вони містять кирилицю, пробіли або спецсимволи (`?`, `()`, `✓`, `✗`).

### Використання

```bash
# Виправити всі діаграми у файлі (шлях прописаний всередині скрипта або передається аргументом)
python scripts/fix_mermaid.py
```

## Повний робочий процес (AI + Scripts)

Найкращий спосіб гарантувати роботу всіх діаграм — це циклічний процес:

### Крок 1: Валідація
Запустіть `validate_mermaid.py`, щоб знайти проблемні блоки.
```bash
python scripts/validate_mermaid.py path/to/article.md
```

### Крок 2: Автоматичне виправлення
Якщо знайдено багато помилок, запустіть `fix_mermaid.py` для базового очищення.

### Крок 3: Перевірка ШІ
Якщо помилки залишилися (наприклад, логічні помилки в структурі графа), попросіть ШІ проаналізувати звіт валідатора:
> *"ШІ, проаналізуй звіт `validate_mermaid.py` для цього файлу та виправ синтаксичні помилки в зазначених блоках."*

### Крок 4: Фінальна перевірка
Повторно запустіть валідатор, щоб переконатися, що тепер все `✅ VALID`.

## Поради для написання валідного Mermaid

Щоб скриптам було легше працювати, дотримуйтесь цих правил при створенні діаграм:

1. **Завжди використовуйте лапки** для тексту з кирилицею: `A["Текст"]`.
2. **Уникайте спецсимволів** у ID вузлів (використовуйте прості `A`, `B`, `Node1`).
3. **Для переносів рядків** використовуйте `<br/>` замість `\n`.
4. **Не змішуйте синтаксис**: пам'ятайте, що те, що працює в `graph TD`, може не працювати в `sequenceDiagram`.
