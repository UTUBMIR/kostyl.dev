# Промпт для генерації тестів (Wayground)

## Мета

Проаналізувати наданий навчальний матеріал і на його основі згенерувати інтерактивний тест для платформи [Wayground](https://wayground.com/). Головний результат твоєї роботи — це повністю готовий тест на платформі, досяжний за прямим посиланням.

## Настанови щодо створення тестів

1. **Глибина запитань**: Запитання мають перевіряти не лише запам'ятовування фактів, але й глибоке розуміння та вміння застосовувати знання на практиці. Уникайте занадто банальних питань.
2. **Типи запитань**: Формуйте різноманітні запитання:
    - `Multiple Choice` (одна правильна відповідь)
    - `Checkbox` (декілька правильних відповідей)
    - `Fill-in-the-Blank` (вставити пропущене слово)
    - `Open-Ended` (відкрите запитання)
3. **Кількість**: За замовчуванням створюйте 10-15 запитань (або іншу кількість, якщо специфічно вказано користувачем), що рівномірно покривають ключові концепції всього матеріалу.
4. **Пояснення (Answer explanation)**: Для кожного запитання **обов'язково** додавайте детальне та зрозуміле пояснення того, чому саме ця відповідь є правильною. Це ключовий елемент навчання.

## Повний флоу: від матеріалу до готового тесту

Весь процес складається з **двох послідовних етапів**. Обидва виконуються автоматично скриптами у папці `scripts/`.

---

### Етап 1 — Генерація xlsx та автоматичний імпорт на Wayground

**Крок 1.1 — Генерація JSON**

Збережи згенеровані питання у тимчасовий `.json` файл. Формат JSON — масив об'єктів зі структурою колонок (детально нижче).

**Крок 1.2 — Генерація xlsx**

Виконай команду в терміналі:
```
python3 wayground_exporter.py <шлях_до_json> -o <фінальний_шлях_xlsx>
```
Скрипт знаходиться в корені проєкту. Фінальний `.xlsx` зберігай у папці `tests/`, відтворюючи структуру папок оригінальних матеріалів. Після успішного створення xlsx видали тимчасовий JSON.

**Крок 1.3 — Автоматичний імпорт та публікація на Wayground**

Завантаж готовий `.xlsx` безпосередньо на платформу через скрипт:
```
python3 scripts/wayground_import.py <шлях_до_xlsx> --name "Назва тесту"
```
Скрипт автоматично:
- логіниться (credentials з `scripts/.env`)
- завантажує xlsx на S3
- парсить питання
- створює порожній тест
- додає всі питання
- **публікує тест** (за замовчуванням увімкнено)

На виході — `quiz_id` і пряме посилання на редактор: `https://wayground.com/admin/quiz/<quiz_id>/edit`.

Якщо потрібна лише чернетка (наприклад, щоб ще додати зображення до варіантів відповідей перед публікацією):
```
python3 scripts/wayground_import.py <шлях_до_xlsx> --name "Назва" --no-publish
```

> **Назва та місце збереження**: Фінальний файл `.xlsx` має точно збігатися з назвою матеріалу і зберігатися у спеціальній папці `tests/` у корені проєкту (відтворюючи структуру папок оригінальних матеріалів).

---

### Етап 2 — Додавання зображень до варіантів відповідей (якщо потрібно)

Імпорт xlsx **не підтримує** зображення у варіантах відповідей — їх треба додавати окремим скриптом після імпорту.

**Крок 2.1 — Автоматична генерація (рекомендовано)**

Якщо варіанти відповідей містять блоки коду, скрипт автоматично згенерує зображення та створить mapping:

```
python3 scripts/generate_answer_images.py <шлях_до_json> <quiz_id>
```

Скрипт автоматично:
- знаходить блоки коду у варіантах відповідей
- рендерить зображення через `silicon` (тема VS Dark+)
- завантажує на Wayground S3
- створює `answer_mapping.json`

**Крок 2.2 — Ручна генерація (альтернатива)**

Якщо автоматичний спосіб не підходить:

а) Переглянь питання:
```
python3 scripts/wayground_add_images.py --quiz-id <quiz_id> --list
```

б) Підготуй зображення через `silicon` (тема VS Dark+) і збережи як `.png` файли.

в) Склади `mapping.json`:

```json
[
  { "question_index": 0, "option_index": 1, "image_path": "images/q1_opt2.png" },
  { "question_index": 3, "option_index": 0, "image_path": "https://files.catbox.moe/abc.png" }
]
```
- `question_index` — індекс питання (0-based, з виводу `--list`)
- `option_index` — індекс варіанта відповіді (0-based)
- `image_path` — шлях до файлу (відносно mapping.json) **або** URL зображення

**Крок 2.3 — Перевірити (dry-run)**

```
python3 scripts/wayground_add_images.py --quiz-id <quiz_id> --mapping mapping.json --dry-run
```

**Крок 2.4 — Застосувати**

```
python3 scripts/wayground_add_images.py --quiz-id <quiz_id> --mapping mapping.json
```
Скрипт завантажує кожне зображення на S3 і PATCH-ає відповідний варіант відповіді.

## Структура колонок файлу `.xlsx`

Файл має суворо відповідати наступній структурі (перший рядок — заголовки колонок, точно як у зразку Quizizz, який підтримується платформою):

1. `Question Text`: Текст запитання. (Обов'язково)
   - **Важливо щодо коду**: Якщо запитання містить приклад коду, оформлюй його як markdown блок коду із зазначенням мови (наприклад: ` ```cpp\n// код\n``` `). Скрипт `wayground_exporter.py` автоматично розпізнає блок, згенерує зображення через `silicon` (тема Visual Studio Dark+), завантажить на `catbox.moe` (постійне зберігання) та запише посилання в `Image Link`. Сам блок коду при цьому видаляється з `Question Text`.
2. `Question Type`: Тип запитання (`Multiple Choice`, `Checkbox`, `Fill-in-the-Blank`, `Open-Ended`, `Poll`, `Draw`). За замовчуванням `Multiple Choice`. (Обов'язково)
3. `Option 1`: Текст 1-го варіанта відповіді. Для `Fill-in-the-Blank` — правильне слово/фраза. (Обов'язково для всіх, крім `Open-Ended` та `Draw`).
4. `Option 2`: Текст 2-го варіанта відповіді (Обов'язково для всіх, крім `Open-Ended`, `Draw`, `Fill-in-the-Blank`).
5. `Option 3`: Текст 3-го варіанта відповіді (Необов'язково).
6. `Option 4`: Текст 4-го варіанта відповіді (Необов'язково).
7. `Option 5`: Текст 5-го варіанта відповіді (Необов'язково).
8. `Correct Answer`: Правильна відповідь:
    - **`Multiple Choice`**: Одна цифра (1–5). Наприклад: `2`.
    - **`Checkbox`**: Номери через кому без пробілів. Наприклад: `1,2,3`.
    - **`Fill-in-the-Blank`, `Open-Ended`, `Poll`, `Draw`**: Порожнє поле (`""`).
9. `Time in seconds`: Час на відповідь у секундах (`30`, `45`, `60`, `120`). За замовчуванням `30`.
10. `Image Link`: Посилання на зображення до запитання. Заповнюється автоматично якщо в `Question Text` є блок коду. Зображення до **варіантів відповідей** через xlsx не підтримуються — для них використовується Етап 2 (`wayground_add_images.py`).
11. `Answer explanation`: Детальне пояснення правильної відповіді. Технічно необов'язково, але **обов'язково** для навчальних цілей.

## Твій алгоритм дій

1. Прочитай і проаналізуй наданий навчальний матеріал.
2. Сформуй якісні запитання, варіанти відповідей, визнач правильні та напиши детальні пояснення.
3. Збережи дані у тимчасовий `.json` файл.
4. Виконай: `python3 wayground_exporter.py <json> -o <xlsx>`. Видали тимчасовий JSON.
5. Виконай: `python3 scripts/wayground_import.py <xlsx> --name "<назва>"` (або `--no-publish` якщо потрібні зображення до публікації).
6. Повідом користувача з посиланням на тест і quiz_id.
7. **Якщо є питання де варіанти відповідей потребують зображень** (наприклад, блоки коду): запусти `python3 scripts/generate_answer_images.py <json> <quiz_id>` для автоматичної генерації. Якщо автоматичний спосіб не підходить — використовуй ручний Етап 2 з mapping.json.
